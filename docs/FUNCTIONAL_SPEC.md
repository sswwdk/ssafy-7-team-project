# LocalHub 기능 명세서

## 1. 문서 개요

| 항목 | 내용 |
|---|---|
| 프로젝트 | LocalHub Team 7 |
| 기준 브랜치 | `master` |
| 기준 커밋 | `372340d` (`map category`) |
| 서비스 목적 | 서울 지역의 장소·축제 정보 탐색과 익명 커뮤니티 제공 |
| 프런트엔드 | Vue 3, Vue Router, Vite |
| 배포 구조 | Netlify 정적 SPA + Netlify Functions |
| 지역 데이터 | `src/data/`의 서울 지역 JSON |
| 사용자 데이터 | 브라우저 `localStorage` |

이 문서는 현재 저장소에서 실제로 확인되는 기능을 기준으로 작성한다. 향후 계획이나 미구현 기능은 포함하지 않는다.

## 2. 사용자 및 권한

| 사용자 구분 | 이용 기능 | 권한 확인 방식 |
|---|---|---|
| 일반 사용자 | 장소·축제 조회, 지도, 검색, 찜, 게시판, 챗봇 | 로그인 없이 이용 |
| 게시글 작성자 | 게시글 수정·삭제 | 작성 시 입력한 비밀번호 |
| 댓글 작성자 | 댓글 수정·삭제 | 댓글 작성 시 입력한 비밀번호 |
| 대댓글 작성자 | 대댓글 수정·삭제 | 대댓글 작성 시 입력한 비밀번호 |

회원가입과 로그인 기능은 없다. 게시판 및 찜 데이터는 다른 사용자나 기기와 공유되지 않는다.

## 3. 화면 구성

| 화면 | 경로 | 주요 기능 |
|---|---|---|
| 메인 | `/` | 추천 장소 슬라이드, 장소 검색, 추천 장소·축제 |
| 대시보드 | `/dashboard` | 서울 날씨, 최근 게시글 |
| 커뮤니티 목록 | `/posts` | 게시글 목록, 검색, 분류 필터 |
| 게시글 작성 | `/posts/new` | 익명 게시글 등록 |
| 게시글 상세 | `/posts/:id` | 추천, 댓글, 대댓글, 수정·삭제 진입 |
| 게시글 수정 | `/posts/:id/edit` | 비밀번호 확인 후 수정 |
| 지도 | `/map` | 지역·장소 종류별 마커와 장소 목록 |
| 축제 | `/festivals` | 월간 캘린더와 축제 상태 필터 |
| Favorite | `/favorites` | 찜한 장소·축제 목록 |
| 404 | 그 외 경로 | 잘못된 경로 안내 |

라우트가 바뀌면 화면 스크롤 위치를 상단으로 초기화한다.

## 4. 공통 기능

### COM-01 상단 메뉴

**기능**

- 메인, 대시보드, 커뮤니티, 지도, 축제, Favorite로 이동한다.
- 로고를 누르면 메인 화면으로 이동한다.
- 다크 모드, 통합 검색, 언어 전환 버튼을 별도 제어 영역에 표시한다.

**완료 조건**

- 각 메뉴를 선택하면 대응하는 Vue Router 경로가 열린다.
- 새로고침하거나 URL로 직접 접근해도 해당 화면이 표시된다.

### COM-02 다크 모드

**기능**

- 밝은 테마와 어두운 테마를 전환한다.
- 저장된 테마가 없으면 운영체제의 색상 모드 설정을 사용한다.
- 선택한 테마를 `localhub.theme`에 저장한다.
- 카드, 필터, 입력창, 커뮤니티 태그, 챗봇 등 주요 UI에 테마를 적용한다.

### COM-03 한국어·영어 전환

**기능**

- 상단 `EN/KO` 버튼으로 언어를 전환한다.
- 선택 언어를 `localhub.language`에 저장하고 HTML `lang` 속성에 반영한다.
- 메뉴, 버튼, 안내 문구, 날짜 형식과 JSON 기반 장소 정보를 변환한다.
- 주요 명소, 서울 자치구, 장소 종류는 등록된 영문명을 사용한다.
- 영문명이 없는 고유명사는 외부 번역 API 없이 로마자 표기한다.
- 영어 모드에서 찜 목록 메뉴는 `Favorite`으로 표시한다.

### COM-04 통합 검색

**입력 대상**

- 장소명
- 주소
- 서울 자치구
- 장소 종류

**동작**

- 검색 버튼을 누르면 검색 패널을 열고 입력창에 포커스한다.
- 한국어 원문 및 생성된 영문 표기를 모두 검색한다.
- 지도 좌표가 있는 결과 중 최대 10개를 표시한다.
- 결과를 누르면 `/map?place={장소 ID}`로 이동한다.
- 지도 화면에서 해당 장소의 지역·종류 필터와 마커 정보창을 자동으로 연다.
- `Esc`, 패널 외부 클릭 또는 닫기 버튼으로 패널을 닫는다.
- 검색어가 없거나 결과가 없으면 각각 안내 문구를 표시한다.

### COM-05 반응형 UI

- 화면 폭에 따라 상단 메뉴, 카드 그리드, 필터와 입력 영역을 재배치한다.
- 지도 필터는 모바일에서 접고 펼칠 수 있다.
- 챗봇은 우측 하단 플로팅 버튼으로 제공한다.

## 5. 메인 화면

### MAIN-01 장소 빠른 검색

- 장소 검색어를 입력하면 최대 6개의 결과를 표시한다.
- 검색 결과에는 종류 아이콘, 장소명, 종류와 자치구를 표시한다.
- 검색 버튼은 첫 번째 결과의 지도 화면으로 이동한다.
- 경복궁, 한강, 축제 추천 검색어를 제공한다.
- 언어 전환 시 추천 검색어와 검색 결과도 현재 언어에 맞게 표시한다.

### MAIN-02 추천 장소 슬라이드

- 청계천, 경복궁, 롯데월드타워 서울스카이, 국립중앙박물관, 서울숲을 우선 표시한다.
- 장소 데이터가 없으면 이미지가 있는 다른 장소를 대체 표시한다.
- 약 4.5초마다 자동으로 다음 장소를 표시한다.
- 좌우 화살표와 하단 페이지 버튼으로 직접 이동할 수 있다.
- 마우스가 슬라이드 위에 있는 동안 자동 재생을 정지한다.
- 슬라이드를 누르면 해당 장소의 지도 마커로 이동한다.

### MAIN-03 장소 종류 탐색

- 관광지, 문화시설, 축제공연행사, 여행코스, 레포츠, 숙박, 쇼핑 카드를 제공한다.
- 카드에 각 장소 종류의 데이터 개수를 표시한다.
- 종류 카드를 선택하면 `/map?category={종류 코드}`로 이동한다.

### MAIN-04 추천 장소 및 축제

- 이미지가 있는 장소를 장소 종류별로 선별해 카드로 표시한다.
- 현재 달에 포함되는 축제를 우선 추천한다.
- 현재 달 축제가 없으면 이미지가 있는 전체 축제 중 일부를 표시한다.
- 장소와 축제 카드에서 관련 상세 화면으로 이동한다.

### MAIN-05 커뮤니티 진입

- 서울 지역정보 공유 안내와 커뮤니티 이동 버튼을 제공한다.

## 6. 대시보드

### DASH-01 서울 날씨

- Open-Meteo의 서울 좌표 데이터를 사용한다.
- 현재 기온, 체감온도, 최고·최저기온, 습도, 풍속, 강수확률을 표시한다.
- 날씨 코드와 주야간 여부에 따라 상태 문구 및 아이콘을 결정한다.
- 브라우저 탭이 다시 활성화되면 날씨를 즉시 갱신한다.
- 로딩, 오류, 재시도 상태를 제공한다.

### DASH-02 최근 게시글

- 현재 브라우저에 저장된 최근 커뮤니티 게시글을 표시한다.
- 게시글이 없으면 빈 상태 안내를 표시한다.
- 전체 커뮤니티 화면으로 이동하는 링크를 제공한다.

## 7. 지도 및 장소 목록

### MAP-01 Kakao 지도

- 환경변수 `VITE_KAKAO_MAP_API_KEY`로 Kakao Maps SDK를 로드한다.
- 기본 지도 중심은 서울시청 인근으로 설정한다.
- API 키 누락, SDK 로드 실패, 지도 영역 누락 시 오류를 표시한다.
- 유효한 위도와 경도가 있는 데이터만 마커로 표시한다.

### MAP-02 지역 필터

- 서울 25개 자치구를 복수 선택할 수 있다.
- `전체 서울`을 누르면 선택된 지역을 초기화한다.
- 선택된 지역에 해당하는 마커와 하단 장소 목록만 표시한다.
- 선택된 첫 지역을 기준으로 지도 중심을 이동한다.
- 필터 제목 아래 현재 선택된 지역 요약을 표시한다.
- 모바일에서 지역 필터를 접거나 펼칠 수 있다.

### MAP-03 장소 종류 필터

- 관광지, 문화시설, 축제공연행사, 여행코스, 레포츠, 숙박, 쇼핑을 복수 선택한다.
- 장소 종류마다 서로 다른 마커 및 카드 색상을 사용한다.
- `전체` 버튼으로 모든 종류를 표시하거나 다시 눌러 선택을 해제한다.
- 기본 상태는 종류 미선택이며 지도 마커와 장소 목록을 표시하지 않는다.
- 제목 아래 현재 선택된 종류 요약 또는 `선택 없음`을 표시한다.
- 모바일에서 종류 필터를 접거나 펼칠 수 있다.

### MAP-04 마커 정보창

- 마커를 누르면 장소 이미지, 이름, 주소가 포함된 정보창을 표시한다.
- 다른 마커를 선택하면 기존 정보창을 닫는다.
- 선택한 마커의 정보창이 잘리지 않도록 지도 중심보다 약간 아래에 마커를 배치한다.

### MAP-05 장소 목록 및 목록 검색

- 지도에 적용된 지역·종류 필터를 하단 장소 목록에도 동일하게 적용한다.
- 카드형과 목록형을 전환한다.
- 장소명, 주소, 자치구, 종류를 현재 목록에서 추가 검색한다.
- 검색 결과 수와 필터 전체 개수를 표시한다.
- 이미지 로드 실패 시 `이미지 없음`을 표시한다.
- 종류 미선택, 검색 결과 없음, 필터 결과 없음 상태를 구분해 안내한다.

### MAP-06 목록에서 마커 보기

- `지도에서 보기`를 누르면 해당 장소 마커의 정보창을 연다.
- 지도 영역으로 자동 스크롤하고 대상 위치로 지도를 이동한다.

### MAP-07 딥링크 처리

- `/map?place={장소 ID}`는 대상 장소의 자치구와 종류를 자동 선택한다.
- 지도 생성 후 대상 마커의 정보창을 열고 지도 영역을 화면 중앙으로 이동한다.
- `/map?category={종류 코드}`는 해당 장소 종류를 자동 선택한다.

## 8. 축제 캘린더

### FEST-01 실시간 정보

- 현재 날짜와 시간을 표시하며 30초마다 갱신한다.
- 서울의 현재 날씨를 간략하게 표시한다.
- 브라우저 탭이 활성화되면 날씨를 다시 조회한다.

### FEST-02 월간 달력

- 한 달을 6주, 총 42개 날짜 칸으로 구성한다.
- 이전 달, 다음 달, 오늘 이동 버튼을 제공한다.
- 오늘, 선택된 날짜, 현재 달 밖의 날짜를 시각적으로 구분한다.
- 날짜별 축제를 최대 2개 표시하고 나머지는 추가 개수로 표시한다.

### FEST-03 상태 필터

- 현재 날짜가 시작일과 종료일 사이면 `진행 중`으로 분류한다.
- 시작일이 현재 날짜 이후이면 `예정`, 종료일이 이전이면 `종료`로 분류한다.
- `진행 중`은 현재 진행 중인 축제만 표시한다.
- `예정`은 예정 축제만 표시하고 가장 가까운 예정 축제의 월로 이동한다.
- `{월} 전체`는 선택 월과 일정이 하루라도 겹치는 모든 축제를 표시한다.

### FEST-04 축제 일정 목록

- 선택 조건에 해당하는 축제를 시작일 순으로 표시한다.
- 이미지, 진행 상태, 기간, 이름, 주소와 설명을 제공한다.
- 결과가 없으면 일정 없음 안내를 표시한다.
- 각 축제에서 찜을 추가하거나 해제한다.

## 9. Favorite

### FAV-01 찜 저장

- 지도 장소와 축제 일정에서 찜 상태를 전환한다.
- `장소 종류 코드 + 장소 ID`를 고유 키로 사용해 중복 저장을 방지한다.
- 찜 목록은 `localhub.favorites.v1`에 저장한다.

### FAV-02 찜 목록

- 저장한 장소와 축제를 최근 찜한 순서로 표시한다.
- 이미지, 이름, 종류, 주소와 축제 기간을 표시한다.
- 항목 종류에 따라 지도 또는 축제 화면으로 이동한다.
- 목록에서 찜을 해제할 수 있다.
- 찜한 항목이 없으면 이용 안내를 표시한다.

## 10. 커뮤니티

### POST-01 게시글 목록

- 게시글을 최신 작성 순서로 표시한다.
- 분류, 작성 시각, 제목, 본문, 추천 수와 댓글 수를 표시한다.
- 댓글 수에는 일반 댓글과 대댓글을 모두 포함한다.
- 제목·내용 검색과 카테고리 필터를 동시에 적용한다.
- 카테고리는 자유, 관광, 맛집, 축제를 제공한다.

### POST-02 게시글 작성

| 입력값 | 조건 |
|---|---|
| 카테고리 | 필수 선택 |
| 제목 | 필수, 최대 80자 |
| 내용 | 필수, 최대 3,000자 |
| 비밀번호 | 필수, 4~20자 |

- 등록 완료 후 생성된 게시글 상세 화면으로 이동한다.
- 게시글을 브라우저 `localStorage`에 저장한다.

### POST-03 게시글 상세·수정·삭제

- 분류, 작성 시각, 제목, 내용과 최종 수정 시각을 표시한다.
- 수정 시 작성 비밀번호를 확인한 후 카테고리, 제목, 내용을 갱신한다.
- 삭제 버튼은 비밀번호 확인 모달을 연다.
- 비밀번호가 일치하면 삭제 후 목록으로 이동한다.
- 비밀번호 불일치 또는 게시글 미존재 시 오류를 표시한다.

### POST-04 추천

- 엄지 아이콘을 사용하는 추천 버튼을 제공한다.
- 다시 누르면 추천을 취소한다.
- 추천 시 개수를 1 증가시키고 취소 시 1 감소시키며 0 미만으로 내려가지 않는다.
- 추천 수는 댓글 수와 다른 아이콘 및 스타일로 구분한다.

### POST-05 댓글

- 기존 댓글 목록을 새 댓글 작성창 위에 표시한다.
- 댓글 내용은 필수이며 최대 3,000자다.
- 댓글 비밀번호는 필수이며 4~20자다.
- 작성 비밀번호가 일치하면 댓글을 수정하거나 삭제한다.
- 수정된 댓글에는 수정 시각을 표시한다.

### POST-06 대댓글

- 각 댓글에 답글을 작성할 수 있다.
- 내용과 비밀번호 조건은 댓글과 동일하다.
- 작성 비밀번호가 일치하면 대댓글을 수정하거나 삭제한다.
- 댓글이 삭제되면 포함된 대댓글도 함께 삭제된다.

## 11. AI 챗봇

### CHAT-01 챗봇 UI

- 모든 화면 우측 하단에 챗봇 버튼을 표시한다.
- 버튼으로 패널을 열고 닫는다.
- 질문 입력, 전송, 로딩, 오류 및 대화 초기화 기능을 제공한다.
- 사용자 메시지는 하단으로 이동하고 AI 답변은 답변 시작 위치가 보이도록 스크롤한다.

### CHAT-02 관련 데이터 선검색

- 질문에 관련된 지역 JSON과 최근 게시글을 브라우저에서 먼저 검색한다.
- 관광지 질문은 자치구를 인식해 관광지 최대 10개를 사용한다.
- 축제 질문은 특정 연월, 이번 달, 다음 달과 자치구를 인식한다.
- 일반 질문은 지역정보 최대 8개와 최근 게시글 최대 5개를 사용한다.
- 전체 JSON이 아닌 검색된 일부 데이터만 AI 문맥으로 전달한다.

### CHAT-03 서버리스 AI 요청

- `/.netlify/functions/chat`으로 요청한다.
- Netlify Function이 서버 환경변수 `OPENAI_API_KEY`를 사용해 OpenAI Responses API를 호출한다.
- 모델은 `OPENAI_MODEL`이 있으면 해당 값을, 없으면 `gpt-5-mini`를 사용한다.
- Function 요청 제한은 25초, 클라이언트 챗봇 제한은 20초다.
- 환경변수 누락, 잘못된 요청, 시간 초과와 API 연결 실패를 오류로 처리한다.
- 챗봇 답변의 장소 링크를 누르면 지도 화면의 해당 마커로 이동한다.

> 한국어·영어 전환 및 JSON 영문 표기는 OpenAI API를 사용하지 않는다. OpenAI API는 챗봇 답변에만 사용한다.

## 14. 예외 및 빈 상태

- 잘못된 경로는 404 화면으로 처리한다.
- 존재하지 않는 게시글은 상세·수정 화면에서 안내한다.
- 게시글·댓글·대댓글의 비밀번호 불일치를 오류로 안내한다.
- 지도 키 누락과 SDK 로드 실패를 지도 영역에 표시한다.
- 지도 종류 미선택, 검색 결과 없음, 필터 결과 없음을 구분한다.
- 축제 일정이 없는 날짜에는 빈 일정 안내를 표시한다.
- 날씨 요청 실패 시 재시도 기능을 제공한다.
- 챗봇 요청 중에는 중복 전송을 막고 로딩 상태를 표시한다.

## 15. 제약 및 주의사항

- 게시글, 댓글, 추천과 찜은 서버 DB가 아닌 현재 브라우저에만 저장된다.
- 브라우저 저장소를 삭제하면 사용자 생성 데이터와 화면 설정이 사라진다.
- 비밀번호는 교육용 요구사항에 따라 `localStorage`에 암호화 없이 저장되므로 실제 중요 정보에 사용하면 안 된다.
- Kakao JavaScript 키는 브라우저에 포함되므로 Kakao Developers에서 허용 도메인을 제한해야 한다.
- 챗봇 API 키는 프런트엔드 환경변수가 아닌 Netlify 서버 환경변수로 관리해야 한다.
- 정적 JSON을 수정하면 다시 빌드하고 배포해야 한다.
