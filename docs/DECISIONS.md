# 기술 결정 기록

## DEC-001 프런트엔드 단일 SPA

- 결정: 백엔드 서버와 Render 배포를 사용하지 않는다.
- 근거: 공통 규칙에서 Vue 3 + Vite 기반 정적 SPA와 프런트 OpenAI 직접 호출을 명시한다.
- 영향: 서버 인증, 서버 DB, 사용자 간 게시글 공유는 범위에서 제외한다.

## DEC-002 localStorage 접근 단일화

- 결정: 게시글 저장과 비밀번호 비교는 `postStorageService.js`에서만 처리한다.
- 저장 키: `localhub.posts.v1`
- 영향: 데이터 구조나 저장 키 변경 시 팀 전체 공유가 필요하다.

## DEC-003 OpenAI 호출 단일화

- 결정: API 요청은 `openaiService.js`에서만 수행한다.
- 방식: Responses API, 타임아웃, 오류 처리, 빈 응답 검증을 한 곳에서 처리한다.
- 보안: 교육용 제한 키만 사용하고 `.env`를 저장소에 포함하지 않는다.

## DEC-004 컨텍스트 선별

- 결정: 전체 JSON을 매 요청마다 전송하지 않는다.
- 방식: 질문 키워드로 지역정보 최대 8건, 게시글 최대 5건을 선별한다.
- 영향: 검색 품질은 실제 JSON 필드 매핑과 동의어 처리에 따라 개선한다.

## DEC-005 선정 권역

- 결정: 실습 기획표를 기준으로 서울을 임시 선정한다.
- 후속: 팀 합의가 다르면 `constants/region.js`, 데이터 파일, 화면 문구를 함께 변경한다.
